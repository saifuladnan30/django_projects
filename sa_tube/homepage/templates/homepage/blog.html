Discuss the scope of var, let, and const!

VAR :

  • Function Scope : Variables declared with var are function-scoped. They are only accessible within the function where they are defined.
  • Hoisting : Variables declared with 'var' are hoisted to the top of their containing function or the global scope. This allows you to use them before they are declared, but they will be initialized with undefined until their actual declaration.

LET :

  • Block Scope : Variables declared with 'let' are limited to the block, statement, or expression where they are defined.
  • No Hoisting : Unlike 'var', variables declared with 'let' are hoisted to the top of their block but are not initialized. They remain in the "temporal dead zone" until the actual declaration.

CONST :

  • Variables declared with 'const' are accessible only within the block where they are defined, and they cannot be reassigned after initialization. While 'const' provides block-scoping like 'let', it also enforces immutability for the variable itself, meaning you can't reassign a new value to it once it's been declared.
Tell us the use cases of null and undefined!

null :

  • Represents the intentional absence of any object value or value in general.
  • Used as a placeholder or initial value that might be assigned a value later.
  • Returned by functions or methods to indicate the absence of a meaningful result.

undefined :

  • Automatically assigned to variables that have been declared but not assigned a value.
  • Indicates the default value for function parameters and variables.
  • Results from attempting to access properties in objects or elements in arrays that do not exist.
  • Returned by functions that do not explicitly return a value.
What do you mean by REST API?

REST API (Representational State Transfer - Application Programming Interface):

  • A set of rules and conventions for building and interacting with web services.
  • Adheres to the principles of the REST architectural style.
  • Enables communication and data exchange between client and server applications over the web.
  • Uses standard HTTP methods (GET, POST, PUT, DELETE) for operations on resources.
  • Emphasizes stateless communication and resource-based interactions.
  • RESTful web services follow these principles to create scalable, maintainable, and interoperable systems on the World Wide Web.
